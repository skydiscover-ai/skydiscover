## Overview

The ADRS (AI-Driven Real-world Systems) benchmarks apply SkyDiscover to real computer systems problems. These are not toy problems — they model production workloads including multi-cloud data transfer, Mixture-of-Experts inference, GPU model placement, and database query optimization.

## CloudCast: Multi-Cloud Data Transfer

**Path:** `benchmarks/ADRS/cloudcast/`

Optimize broadcast topology for multi-cloud data transfer to minimize cost. The algorithm must route data partitions from a source region to multiple destination regions across cloud providers (AWS, Azure, GCP), respecting bandwidth and cost constraints.

### Initial Program

The initial program uses Dijkstra's shortest path as a baseline:

```python
# EVOLVE-BLOCK-START
import networkx as nx
from typing import Dict, List


def search_algorithm(src, dsts, G, num_partitions):
    h = G.copy()
    h.remove_edges_from(list(h.in_edges(src)) + list(nx.selfloop_edges(h)))
    bc_topology = BroadCastTopology(src, dsts, num_partitions)

    for dst in dsts:
        path = nx.dijkstra_path(h, src, dst, weight="cost")
        for i in range(0, len(path) - 1):
            s, t = path[i], path[i + 1]
            for j in range(bc_topology.num_partitions):
                bc_topology.append_dst_partition_path(dst, j, [s, t, G[s][t]])

    return bc_topology


class BroadCastTopology:
    def __init__(self, src: str, dsts: List[str], num_partitions: int = 4,
                 paths: Dict[str, dict] = None):
        self.src = src
        self.dsts = dsts
        self.num_partitions = num_partitions
        if paths is not None:
            self.paths = paths
        else:
            self.paths = {
                dst: {str(i): None for i in range(num_partitions)}
                for dst in dsts
            }

    def set_num_partitions(self, num_partitions: int):
        self.num_partitions = num_partitions

    def set_dst_partition_paths(self, dst, partition, paths):
        self.paths[dst][str(partition)] = paths

    def append_dst_partition_path(self, dst, partition, path):
        partition = str(partition)
        if self.paths[dst][partition] is None:
            self.paths[dst][partition] = []
        self.paths[dst][partition].append(path)
# EVOLVE-BLOCK-END
```

The evaluator replays real expert-load traces (288 replicas, 8 groups, 32 GPUs, 4 nodes), simulates inference load distribution, and measures balancedness:

## Key Concepts

### System Constraints

Unlike math benchmarks, systems problems have hard constraints:
- Graph connectivity (CloudCast)
- GPU memory limits (Prism)
- Data partitioning rules (EPLB)
- Solutions that violate constraints get a zero score.

### Multi-Objective Optimization

Systems benchmarks often balance multiple objectives:
- Cost vs. latency (CloudCast)
- Balancedness vs. speed (EPLB)
- Memory vs. throughput (Prism)

The evaluator combines these into a single `combined_score`.

### Simulation-Based Evaluation

Each evaluator runs a simulator to measure real-world impact:
- CloudCast simulates network transfer costs
- EPLB replays production MoE inference traces
- Prism models GPU memory and cache behavior

### Real Workloads

All ADRS benchmarks use real-world data:
- Cloud pricing and throughput profiles (CloudCast)
- Expert load histories from production serving (EPLB)
- GPU memory and bandwidth measurements (Prism)
